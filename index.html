Salesforce Flow Builder — automated workflows
When student → Qualified
Trigger: Contact status picklist changes to "Qualified"
Auto-create Match Review task and assign to staff queue
📋
When session → Completed
Trigger: Session record status = "Completed"
Send feedback survey to student, patient, and supervisor via email
📧
When university → Pilot
Trigger: Account stage changes to "Pilot"
Notify onboarding team and create setup task checklist
🔔
Awaiting match — 7+ days
Trigger: Scheduled flow checks daily — match date null AND status = "Awaiting"
Alert administrator to intervene and escalate for manual match review
⚠️
University renewal window
Trigger: Renewal date within 30 days
Create renewal task for account manager with contract summary
📅
Feedback approved
Trigger: Testimonial approval flag = True on Feedback record
Flag testimonial for marketing review and archive with session record

Profiles & permission sets

HIPAA requirements checklist
Business Associate Agreement (BAA)
Salesforce Shield / Health Cloud edition
Audit trail (login + data access)
Data encryption at rest
MFA enforced for all users
Data de-identification approach
Patient object uses internal ID
PHI not stored on Match record
Feedback testimonials reviewed before publish
Session notes access controlled
⚠️ Salesforce is not HIPAA-compliant by default. VPEd must sign a BAA and use the correct edition (e.g. Health Cloud or Salesforce Shield) before going live with patient data.
